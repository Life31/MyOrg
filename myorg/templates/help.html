Общие сведения
Ресурс регистрации заявок состоит из:
1) Страница регистрации пользователя;
2) Страница создания/редактирования заявки;
3) Страница календарей;
4) Страница стенда;
5) Страница пользователя;
6) Страница заявки.
Функционал ресурса в части создания и редактирования заявок доступен только авторизованным пользователям.
Навигация по ресурсу происходит с помощью бокового меню.
1. Регистрация
Для регистрации на ресурсе необходимо:
1) Нажать на ссылку «Регистрация» в правом верхнем углу ресурса;
2) Заполнить форму регистрации и нажать кнопку «Зарегистрироваться».
Имя пользователя и адрес электронной почты должны соответствовать указанным в почте Outlook.
Пример:«nikolay.emelyanov» и «nikolay.emelyanov@ic.irkut.com»
2. Создание заявки
Для создания новой заявки необходимо в меню навигации, в разделе «Создать заявку» выбрать нужный стенд:
SIB
FIB
ACIB
NUC
ISIB
FIB BM
FVB
Далее, заполнить форму создания заявки и нажать кнопку «добавить».
Если нужное подразделение-инициатор отсутствует, необходимо отправить запрос администраторам ресурса на
добавление соответствующего подразделения.
В случае некорректного заполнения формы, заявка не будет создана, а пользователь получит соотвествующее предупреждение.
После создания заявки:
1) Заявке присваивается статус «Не подтверждена»;
2) Заявка отображается на странице заявки;
3) Заявка отображается на странице соответствующего пользователя;
4) Заявка отображается на странице соответствующего стенда;
5) Администраторы ресурса получают почтовое уведомление о создании новой заявки;
6) В случае, если интервал времени испытаний соответствующей заявки, пересекается с интервалом времени уже созданной заявки,
пользователь получает предупреждение.
Пользователь может отредактировать заявку или договориться с автором приоритетной заявки.
Администратор может подтвердить заявку, отклонить и, при необходимости, оставить комментарий на странице заявки.
После подтверждения заявки администратором, она отображается в календарях.
Пользователю приходит почтовое уведомление о смене статуса заявки.
Доступ к содержимому заявки можно получить:
1) Нажав на соответствующую заявку на странице календарей;
2) Нажав на соответствующую заявку на странице пользователя;
3) Нажав на соответствующую заявку на странице стенда;
4) Перейдя по ссылке в почтовом уведомлении.
При редактировании уже подтвержденной заявки ее статус с «Подтверждена» автоматически меняется на
«Не подтверждена» и администраторы ресурса повторно получают почтовое уведомление о создании новой заявки.
3. Календари
Для перехода на страницу календарей необходимо в меню навигации, в разделе «Календари» выбрать соответствующий календарь:
День Неделя
Месяц Год
SIB
FIB
ACIB
NUC
ISIB
FIB BM
FVB
Основная цель календарей - сгруппировать и отобразить заявки в рамках дня/недели/месяца или стенда.
Основной функционал – переход из календаря на страницу заявки по клику.

FIB
{% if acib_count > 0 %}
ACIB({{acib_count}})
{% else %}
ACIB

Основная цель страницы стендов - отображение перечня заявок, отфильтрованных в соответствии с выбранным стендом.
Основной функционал:
1) переход со стенда на страницу заявки по клику;
2) переход на страницу автора заявки (Заявитель) по клику;
3) фильтрация списка заявок по статусу.
5. Страница пользователя
Для перехода на страницу пользователя необходимо в правом верхнем углу нажать на ссылку:
{% if user.is_authenticated %} {{ user.username }} {% else %}Имя пользователя{% endif %}
Ссылка доступна только авторизованным пользователям.
Основная цель страницы пользователя - отображение перечня заявок, отфильтрованных в соответствии с выбранным автором.
Основной функционал:
1) переход со страницы пользователя на страницу стенда по клику;
2) переход со страницы пользователя на страницу заявки по клику;
3) фильтрация списка заявок по статусу.
6. Страница заявки
Доступ к странице заявки можно получить:
1) Нажав на соответствующую заявку на странице календаря;
2) Нажав на соответствующую заявку на странице пользователя;
3) Нажав на соответствующую заявку на странице стенда;
4) Перейдя по ссылке в почтовом уведомлении.
Основная цель страницы заявки - отображение информации о заявке.
Основной функционал:
1) Редактирование заявки (автором заявки);
2) Удаление заявки (автором заявки);
3) Использование заявки в качестве шаблона для новой заявки;
4) Комментирование заявки.
5) Управление реле питания стенда активной заявки (автором заявки).
